### Mathematica, 它好就好在...

那我为什么建议大学生都下个Mathematica来玩玩呢.

第一当然是因为它**简 单 啊**!

我知道很多学生学习软件最大的阻力是什么捏, 是 懒 啊. 你想想你买到的时候有没有认真阅读完说明书? [^2] 用化学试剂前查过MSDS[^3]了吗? 甚至看小说都不想看设定复杂的. 一个软件的学习时间要是太长了必然会提高其门槛. 

如果和学高等数学相比, Mathematica的入门难度就像数林檎一样简单. 如果指导得当一小时就能上手.我们用一段代码来举例子:

```mathematica
Plot3D[{x^2 + y^2}, {x, -2, 2}, {y, -2, 2}, 
 ColorFunction -> "RustTones"]
```

函数名简单,使用直观的类似英文的函数名称和一致明了的设计. `Plot3D[]`直译「绘图 3D」,就是画3D图形嘛. `ColorFunction`很明显就是「给函数上色的意思」.极佳的可读性降低了门槛, 编写起来就像呼吸一样轻松.

公式阅读简单.比如说在别的语言里分数只能委屈成除法,但是在Mathematica中只需要<kbd>Ctrl</kbd>+<kbd>/</kbd>就能实现分式,可以直接使用积分号来表示积分, 甚至连矩阵都能直接表示:

![mathematica矩阵演示](star/image-20220219192238345.png)

如此贴近自然书写格式让人无法拒绝. 这意味着只需要把你书上的题目抄到屏幕上就能得出结果. 在检查答案时可以纵览全局, 而不用盯着一堆括号瞎改.

第二是因为**立绘好看**啊. Mathematica不仅操作容易,直观易得,而且美观.Mathematica在科研软件中的审美水平独为一档,<kbd>Enter</kbd>+<kbd>Shift</kbd>按一下就能执行得到配色美观, 完美展现计算美学的结果.

第三是因为它**啥都能干**功能多多.要做傅里叶变换?冇问题, FourierTransform就行. 级数展开? 来来来, Series函数直接展.各种功能已经被封装好了在一个个函数里面,万能的<kbd>F1</kbd>键会告诉你所有函数的用法.你甚至能用Mathematica查看当天的股市.

![Mathematica的功能们](star/image-20220220111730071.png)

那我为什么说Mathematica有助于学习呢? 你想啊, 要是我在学习微分几何时就用Mathematica在曲面上画出迪潘指标线不就秒懂了. 要是能画出图像也能很快理解有的函数为什么有偏导而不可微. 交作业前也能仔细看一看有没有地方算错了, 还能逐步验算找到错误点, 真实的能提高效率. 而不必如苦行僧一般刷题. 



## Markdown

>   真正的次世代写作方法. 

由于学习的需要, 我时不时会电脑上写点东西来记录自己的学习过程. 至于为什么不用手写呢, 呃...你懂的...就是写的字有点不太雅观. 面对如狂草般的笔记谁都不想捏着鼻子看下去吧. 那么我需要的软件至少满足以下要求:

1.   能方便的打公式. 对学理科的来说这个是刚需;
2.   易上手. 要是还需要看十几节课那种学习成本太高;
3.   简洁美观. 设置项太多了人家根本看不过来啦;
4.   方便编辑. 能全用键盘操作最好, 鼠标点点点会降低效率;
5.   能本地保存, 不依赖特定软件. 要是你厂商倒了我岂不是就看不了我的文档了. 而且这样有一种知根知底,能完全支配的感觉. 

Word就这样彻底沦为了排版软件: 在我每次编辑公式时都想把公式编辑器砸了. 至于插入图片的时候——它总能出现在我意想不到的地方,还有各种花花绿绿的设置项. txt? 完全没有格式的话我看不出来重点和题目的区分,而且太简陋了.  Notion虽然是全平台, 但数据始终在云端, 公司垮了我上哪找笔记. Tex呢?太复杂了学习成本高. 而且体积又大,环境又难装. (为什么要把排版的用来做笔记啊kora).

综上所述, 能满足我全部要求的就是Markdown(其实是换个法子谈优点). 

这里有几个推荐的Markdown编辑器:

### Typora 

仅beta版免费. 但就是beta版本的它在Windows上的表现已经几乎是完美的:

-   所见即所得的编辑方式
-   公式也能所见即所得
-   立绘(软件界面)好看
-   极简的设计风格

缺点就是文字太多时(10000+)会比较卡.

暂时没找到优质的替代品. 

### VS code

这种事情宇宙最强编辑器怎么可能不行呢. 事实上具有丰富插件的VS code可以有非常顺滑的体验:

-   **可以有**所见即所得的编辑方式
-   **可以**让公式也所见即所得
-   立绘(软件界面)好看
-   任你自己造的设计风格, 高度自定义
-   10000+的时候不卡

我写代码的时候就喜欢用这个, 在一个软件上不用切过来切过去的很方便.

### [Markdown Online](https://markdown.com.cn/editor/)

网页上也能编辑啦, 浏览器教徒们可以用:

-   能有多种复制格式

不过在网页上总会担心哪天网站倒闭了...



移动端就不推荐了.(话说为什么会有移动端有生产力的想法)

*严格意义上来说Markdown是富文本标记语言, 推荐的那些才是软件*



## Axmath/MathType

>   拜托, 会latex真的是一件很酷的事

曾几何时我还是狂热的Unicode分子的时候, 我对latex不屑一顾. 我恨不得所有的地方都适配Unicode, 公式?自然得用我们正紫旗儿[^4] 的, 臭写代码的跑我们这儿要饭来了. 直到有一天彻底对Word的Unicode-math公式编辑器大破防: 为了赶完20篇没写完的实验报告和预习报告(疫情期间堆起来的), 我在word里狂写了两天的的公式,很多问题就凸显出来了:

-   word自带的unicode-math编辑器应付少量公式还好, 但是物理学科公式泛滥, 数量一多Unicode-math就会卡;

-   而且unicode-math也没有快捷键能设置, 关键词和latex一样;

-   别的软件不兼容, 换个地方根本输入不了.  

到现在我仍在怀疑微软拿出这个东西的底气在哪(***退钱!), 所见即所得的优势都没有了, 可能唯一的优势就是集成在word里面了不用切换软件(所以我写作也放弃了word).我现在会劝每一个还在用unicode-math公式编辑器的人: 你改悔罢[^5].

---以上是一位Unicode爱好者的破防记录

总之, 为了不吃二遍苦, 受二茬罪, 我抛下脸面去学Latex来编辑公式. 回 头 是 岸, latex流畅的编辑速度与广泛的应用范围让我欢喜, 历史和人民选择它是有道理的, latex发明就是为了解决数学公式排版难题的,(参见:[在LaTeX发明前，人们怎么打印复杂的数学公式？ - zhihu.com](https://www.zhihu.com/question/406922677/answer/1343750748) ),latex渲染的公式也是独一流的.

但是latex也有个缺点: 代码太长. 此处提一个很容易忽略的点: 数学中的诸如$$\sin$$,$$\cos$$这些是需要使用Roman字体的, 而且常数$$\mathrm{i}$$(虚数单位),$$\mathrm{e}$$(自然常数)也是需要Roman字体的,有内置函数的还好, 像常数啥的写起来就很麻烦. 与此同时还有麻烦的多行与矩阵,\\begin{啥啥啥}\\end{啥啥啥} 成串出现, 全手打的话确实会比较慢. 要是关键词能像汉语输入法一样打出前几个就能自动补全, 多行可以直接用快捷键就好了. (说到底就是差一个方便写latex代码的)

**有!**

axmath就能完美解决这种需求. 不仅写latex代码方便, 甚至还带有所 见 即 所 得的喜人功能. 新手可以到处点一下, 熟练的人手都不会离开键盘, 快捷键加联想输入直接快速解决.

mathtype也可以, 但是国内被**苏州思杰马克丁公司**独占代理, 你能找到的嗯嗯...学习版,基本都被替换成了他们公司的版本. 不仅如此, 此公司为国内提供的是**价格更高**但是**阉割功能**并且**不提供新版本**的**非正版**软件(真是罄竹难书). 如果能找到正版渠道也是一种选择.

说到这里为大家提供一下此公司代理的部分软件, 选择时请仔细甄别:

-   FL studio
-   Vegas pro
-   几何画板

-   etc...

互联网的发展是如此翻天覆地, 以至于蒙住了我们的双眼. 我们是否过于沉浸在信息流之中, 忘记了那个仰望星空的自己. 人类靠着智慧之实, 从高耸的珠穆朗玛到黑暗的马里亚纳都留下了印记, 向无限深空送出地球之声(Murmurs of Earth),在遥远的星球上探索生命的可能性. 人类从未放弃过探索.
